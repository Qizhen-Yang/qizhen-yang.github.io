为了省事，同时也能更方便的编写、管理网页，我还是重新用起了 hexo。

安装，初始化

首先安装 node.js。然后使用 npm 安装 hexo。

然后在对应文件夹里初始化 hexo。

就会有这样的目录：

就好了。

修改主题，调整主题

网上有非常多的 hexo 博客主题，我挑了 keep。
使用起来非常简单，把整个主题包下载下来解压放到 /themes 文件夹里，然后在 _config.yml 中将 theme 设置为主题包的文件夹名就可以了。

主题有很多默认的用不到或没使用的功能，很多都能在主题包下的 _config.yml 里修改，比如页首的菜单等。
剩余的一些需要手动修改主题文件，比如 source 文件夹内的样式表，layout 文件夹内的 ejs 模板，languages 文件夹内的语言包等，还是非常方便的。

写文章，生成页面，预览

用 hexo 写文章非常方便，直接在命令行里输入 hexo new "文章标题"，就会自动在 /sources/_posts 下生成带标题和时间信息的 markdown 文件。

添加页面要使用命令 hexo new page "页面标题"，会自动在 /sources 下生成 页面标题 文件夹以及文件夹内的 index.md。
使用 hexo new page 404 生成页面，并在 markdown 文件开头加上 permalink: /404.html 就可以生成 404 页面来替换掉默认的了。

命令 hexo g 可以在 /public 文件夹下生成对应的 html 文件等，hexo clean 可以清理掉这些文件。
hexo s 会在本地搭建服务端来更好的模拟发布后的效果。

发布

经过测试，hexo 官网所说的 hexo deployer git 似乎并没有正常运行，没法实现只将 /public 下的文件上传至 GitHub。

但是还有一种解决方法，那就是直接在 /public 下建立一个 git 仓库，然后将这个仓库发布到 GitHub，除了没法使用 hexo clean 之外没有特别大的区别，写一个两行的命令行脚本也可以实现在生成完之后自动 push 到 GitHub。

最后为了国内访问速度更快一点，将 GitHub 仓库再部署到 vercel 就可以了

如果直接用 vercel 提供的 hexo 会更方便一点，不需要这么多本地的命令行操作，只要在 GitHub 上往 /source 下放 markdown 文件就可以创建页面了，也不用手动生成，唯一的缺点就是每次 markdown 的头上那几行没法自动生成。
